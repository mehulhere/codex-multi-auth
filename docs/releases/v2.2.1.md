# v2.2.1

Patch release. A full deep audit of the v2.2.0 tree (six parallel auditors across
auth, the runtime proxy/rotation, scripts/bins, quota/policy, storage, and the
codex-manager CLI, plus a live Windows/PowerShell repro) surfaced bugs that
escaped pre-release review. The headline fix: the `mcodex` launcher (the v2.2.0
flagship) was a bash script shipped as a Windows bin and could not start on
Windows when a WSL stub shadowed git-bash on PATH; it is now a pure Node launcher
with zero bash dependency.

This release also folds in PR #506: `gpt-5.5` becomes the default live/quota
probe model (legacy Codex fallback chain preserved), Codex-unavailable accounts
are labeled "signed in" rather than "working", and the Codex plugin manifest
ships in the package.

## Install

```bash
npm i -g codex-multi-auth@latest
```

## Launcher

### Fixes

- Rewrote the `mcodex` launcher as a Node entry (`scripts/mcodex.js`). The old
  `#!/usr/bin/env bash` script shipped as a Windows bin; npm's generated
  `.cmd`/`.ps1` shim invoked bare `bash`, so when the WSL stub
  (`System32\bash.exe`) or the WindowsApps app-execution alias resolved before
  git-bash, `mcodex` died with `HCS_E_SERVICE_NOT_AVAILABLE`. The Node launcher
  has zero bash dependency on the default forward path, invokes `tmux`/`watch` as
  argv arrays (no shell interpolation), and degrades with the same friendly
  messages when those POSIX tools are absent.
- Canonicalized the direct-run gate (realpath) so the launcher still runs when
  invoked through an npm-created symlink bin.
- Relayed `SIGTERM`/`SIGINT` to the spawned child so it is never orphaned.

## Auth

### Fixes

- Isolated OAuth concurrent-login state. The local callback server stored the
  authorization `code`/`state` on the shared `http.Server` instance, so two
  logins in one process could cross-bind callback state; capture now lives in
  per-call closures.
- Serialized the local-client-token store's full read-modify-write through its
  write queue and widened the rename retry set to the complete transient lock
  taxonomy (`EBUSY`/`EPERM`/`EAGAIN`/`ENOTEMPTY`/`EACCES`); `lastUsedAt` writes on
  the bearer-verify hot path are debounced so steady-state verification stays
  in-memory.

## Runtime Rotation

### Fixes

- Stripped inbound `cookie` / `proxy-authorization` on both egress paths and
  bounded the proxy's upstream error-body read.
- Closed the `routingMutex="enabled"` selection race: account selection and the
  cursor commit now run inside a single, reentrant mutex acquisition, so
  concurrent requests can no longer read the same cursor and stampede the same
  account. Legacy mode is unchanged.
- Short-circuited per-request storage re-reads on unchanged mtime/size and moved
  the authorization check before path/method discrimination (401 before 404).
- Bucketed a model-less `/codex/responses` request into the codex rotation family
  (via `CURRENT_CODEX_MODEL`) instead of the general `gpt-5.5` family, keeping
  rotation/cooldown/budget accounting correct.

## Storage

### Fixes

- Fixed a storage-transaction deadlock: a flagged-storage recovery (backup
  restore and legacy-blocked-file migration) that ran inside an already-held
  storage lock re-acquired the global mutex and wedged all subsequent
  account/token saves (reachable via the doctor restore flow). Lock ownership is
  now tracked so recovery persists without re-locking.
- Preserved `pinnedAccountIndex`/`affinityGeneration` through the combined
  account+flagged transaction clone, so a doctor restore no longer erases the
  user's manual pin.
- Created secret directories `0o700` (and re-asserted on POSIX) instead of
  relying on the umask, so they are not world-listable.
- Applied the env-path config save's read-modify-write under a cross-process file
  lock (with an owner token and compare-before-unlink release) plus a retried
  `stat` and mtime compare-and-swap, closing the lost-update window.
- Floored fractional indices and coerced `NaN` in `clampIndex`; rejected NUL-byte
  paths and validated stored ids before building filesystem paths.

## Quota & Forecast

### Fixes

- Set `gpt-5.5` as the default live/quota probe model, with the legacy fallback
  chain (`gpt-5.4` → `gpt-5.3-codex` → `gpt-5.2-codex` → `gpt-5-codex`) preserved
  in one shared `QUOTA_PROBE_MODEL_CHAIN` constant so the manager and runtime
  probes cannot drift apart.
- Labeled Codex-unavailable accounts as "signed in" rather than "working": live
  check gained `Codex available` / `signed in only` / `need re-login` counters,
  and such accounts render `!` instead of `✓`.
- Classified the normalized "model is not currently available for this ChatGPT
  account" wording as an entitlement block (not a transient outage) consistently
  across the probe / forecast / report / check surfaces.
- Excluded policy-blocked and token-exhausted accounts from forecast
  recommendations; the forecast now returns no recommendation with a clear reason
  when none are ready.
- Fixed the status-tone precedence so a failed live check whose detail carries a
  quota percentage renders red, not green.

## CLI

### Fixes

- Hardened `--model` parsing across every CLI parser (best, forecast, report,
  fix, integrations, models): a flag-like or whitespace-only value after
  `--model`/`-m`/`--model=` is rejected with a clear "Missing value" error
  instead of silently consuming the next flag.
- Rejected non-integer `workspace`/`switch` indices (e.g. `1.9`, `2abc`) with a
  "must be a positive integer" message instead of silently truncating via
  `parseInt`.
- Rejected a flag-like budget key in `budget check` (e.g. `budget check --json`)
  instead of consuming the flag as the key name.
- Shipped the Codex plugin manifest (`.codex-plugin/plugin.json`) in the package,
  enforced by the pack-budget check via an exact-file requirement (a `.bak`
  sibling can no longer satisfy the gate).
- Read the capability matrix under the entitlement key (matching the write path)
  so it no longer reports every account as supporting every model; clamped
  out-of-range quota percentages; made capability-policy eviction LRU.

## Release Hygiene

### Tests

- Added 40+ regression cases across the fixed surfaces: the Node launcher's
  arg/env parsing, signal relay, and symlink direct-run gate; the storage
  re-entrant-lock and legacy-migration deadlock paths; cross-process config-lock
  ownership; `--model` flag-like/whitespace rejection on every parser; the
  budget-key guard; the quota probe fallback chain; and the live-check
  signed-in-only summary.
- Routed the duplicated quota-probe candidate list and the default-model literal
  through single shared constants (`QUOTA_PROBE_MODEL_CHAIN`, `DEFAULT_MODEL`) to
  reduce future drift, with test assertions bound to those constants.

### Notes

- Two findings are documented as intentional rather than changed: the device-auth
  endpoint's bare 403/404 responses are its non-RFC-8628 "authorization pending"
  signal (the poll already exits at the server deadline), and runtime budgets are
  deliberately soft/eventually-consistent under concurrency.
