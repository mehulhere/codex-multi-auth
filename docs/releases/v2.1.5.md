## New Features
- Added manual-only update notices for installed wrappers so startup can surface available package updates without running `npm install` or mutating global packages. (#465)
- Added runtime shadow-home isolation for Codex goal, log, and state SQLite files so wrapper-launched Codex runs start from clean local runtime state while preserving user config, sessions, plugins, skills, and multi-auth state. (#467)

## Bug Fixes
- Removed package-owned automatic update behavior and detached startup update checks from forwarded Codex launch paths. (#465)
- Fixed `thread/goal/get` proxy failures so recoverable upstream errors and exhausted account pools return `{ "goal": null }` instead of surfacing noisy Codex TUI goal-read errors. (#466)
- Fixed runtime shadow-home startup around large or locked Codex `logs_*.sqlite*` and `state_*.sqlite*` files, stale generated directories, Windows sidecar handling, platform filename casing, and atomic SQLite mirror cleanup. (#467)
- Stabilized update-notice package-root resolution across local Node versions and repo path layouts.

## Documentation
- Updated user docs to point to the explicit manual update command and remove automatic-update expectations from configuration, upgrade, settings, and command references. (#465)
- Added `v2.1.5` release notes and refreshed README/docs release-history links.

## Chores
- Replaced the auto-update checker with the manual update-notice path and removed the old auto-update test surface. (#465)
- Expanded wrapper regression coverage for runtime shadow-home SQLite skip/mirror behavior, sidecar fallback warnings, session-index fast paths, and update-notice root handling. (#465, #467)
- Validated the release with lint, TypeScript, repo hygiene, vendor provenance, build, package budget, audit, dry-run publish, full tests, published-install smoke, upgrade smoke, and real Codex CLI stress.
