## New Features
- Added `codex auth login --device-auth` as the preferred remote and headless OpenAI Codex login path. (#441)
- Added device-code polling, authorization-code exchange through the Codex device redirect URI, and persistence through existing account storage, Codex CLI sync, and active-account state. (#441)

## Bug Fixes
- Rejected conflicting `--device-auth` combinations with `--manual` or `--no-browser` while preserving browser-first, manual callback paste, and no-browser login behavior. (#441)
- Kept device-code poll responses out of logs and account storage while preserving the upstream PKCE verifier exchange boundary. (#441)
