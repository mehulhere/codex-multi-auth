# v2.2.0

Stable release. Promotes the v2.1.13-beta line to stable and adds the `mcodex`
launcher. It bundles the Phase 1 correctness/security audit (#499), the new
`mcodex` launcher + cached statusline (#500), the quota unsupported-model
detail-shape detection (#501/#502), and the hardening surfaced by a whole-tree
stress test (#503). It also carries the cascade OAuth token-invalidation fix, the
multi-workspace support, and the pinned-account 503 diagnostic that shipped
across v2.1.13-beta.0–beta.3.

## Install

```bash
npm i -g codex-multi-auth
```

## Launcher

### Features

- New `mcodex` launcher (#500): launches Codex with a cached status line printed
  before startup (model, reasoning effort, cwd, active account, quota usage,
  plan, cache age). `mcodex --tmux` runs inside a tmux session with mouse
  scrollback; `mcodex --tmux --live-accounts` adds a live `codex-multi-auth list`
  pane; `mcodex --monitor` is monitor-only.
- The status line reads local `quota-cache.json`, `runtime-observability.json`,
  and account storage and never calls OpenAI on launch; it refreshes quota data
  in the background only when the cache is stale (default 10 min,
  `CODEX_MULTI_AUTH_STATUS_QUOTA_REFRESH_INTERVAL_MS`), behind a lock so
  concurrent launches don't double-refresh. Resolves the per-project account pool
  when `perProjectAccounts` is enabled and Codex CLI sync is off. Toggle with
  `CODEX_MULTI_AUTH_STATUSLINE=0`.

### Hardening

- Validated `MCODEX_MONITOR_INTERVAL` / `MCODEX_TMUX_HISTORY_LIMIT` as numeric
  before interpolating them into `watch` / tmux commands (no shell injection).
- `--monitor` / `--live-accounts` fail fast with a clear message when `watch`
  isn't installed instead of spawning a broken pane.
- The status path resolves `~` correctly on Windows (`path.sep`, not a hardcoded
  `/`) and reads the account pool without blocking the event loop.

## Runtime Rotation

### Fixes

- Bound the runtime rotation proxy and local bridge to loopback only with no
  opt-out, and stopped forwarding inbound client credentials (`authorization`,
  `x-api-key`, `cookie`, `proxy-authorization`) upstream alongside the managed
  token; IPv6 loopback (`::1` / `[::1]`) is normalized for both the socket bind
  and the emitted base URL (#499).
- Stripped inbound `cookie` / `proxy-authorization` on both egress paths and
  bounded the proxy's upstream error-body read, previously unbounded on 4xx/5xx
  (#503).

## Storage

### Fixes

- Verified cached Codex instructions with SHA-256: a tampered cache is discarded,
  and a legacy entry with no recorded digest is treated as unverified — never
  fast-path served and never driving a conditional `304` revalidation (#499).
- Validated stored message/part records before using their ids to build
  filesystem paths, quarantining a parseable-but-unsafe id (e.g. `../poison`) or
  a non-numeric `time.created` instead of escaping into a traversal read or
  `NaN` sort; rejected NUL-byte paths in `resolvePath` (#499, #503).
- Made the account store atomic and self-healing (checksummed WAL +
  temp-and-rename; a torn write self-heals on the next read) and retried the
  shared transient-lock taxonomy (EBUSY/EPERM/ENOTEMPTY/EACCES/EAGAIN) across
  WAL/temp writes, temp cleanup, backup copy/rename, quota-cache, flagged-store,
  and export so an antivirus/indexer lock doesn't fail a valid operation or
  strand a secret-bearing temp file (#499).
- Persisted `runtime-observability.json` owner-only (`0o600` / dir `0o700`) on
  POSIX (#503).
- Removed synchronous `Atomics.wait` sleeps from the config load path and the
  logger's directory-creation retry so neither freezes the event loop (#499).

## Quota & Forecast

### Fixes

- Detected an unsupported Codex model from the upstream error `detail` shape (not
  just the nested `error` envelope), so a Codex-gated account surfaces a friendly
  "Codex unavailable" note across the `best` / `forecast` / `report` / live-check
  surfaces instead of leaking the raw upstream text (#501/#502).
- Kept a genuine transient failure mixed into the probe surfacing as the real
  error, never masked behind the friendly note, so a real outage is not hidden
  (#501/#502).
- Bounded prompt and release-metadata fetches by connect+body timeouts that
  actually cancel a stalled body (#499).

## CLI

### Features

- `codex-multi-auth status` / `list` gained `--json` for machine-readable output,
  with a stable shape whether or not accounts are configured (#499).

### Fixes

- Required a strict integer for `switch <index>` (no silent float truncation)
  (#503).
- Masked tokens and emails across log, debug-bundle, and status sinks; the debug
  bundle redacts the home prefix (case- and separator-insensitive on Windows),
  strips credentials from config values, and masks the account id (#499).

## Release Hygiene

### Dependencies

- Bumped `vitest` to `^4.1.8` (dev-only) to clear GHSA-5xrq-8626-4rwp (#503).
