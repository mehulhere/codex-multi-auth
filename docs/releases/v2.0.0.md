## New Features
- Added opt-in runtime Responses rotation with `codex auth rotation enable`, `disable`, `status`, `bind-app`, and `unbind-app`. (#438)
- Added the `codex-multi-auth-runtime-proxy` provider path for forwarded official Codex CLI/app Responses requests. (#438)
- Added quota, auth-refresh, network, and upstream-server failover across managed accounts without requiring constant session restarts. (#438)
- Added shadow `CODEX_HOME` handling, sync-back for refreshed official Codex state, reversible packaged Codex app binding, and runtime app-helper support for wrapper-launched `codex app` sessions. (#438)

## Bug Fixes
- Fixed proxy response forwarding so decoded upstream responses do not retain stale `content-encoding`. (#438)
- Hardened shadow-home sync locking so transient owner-write failures do not leave orphaned locks, while keeping runtime rotation localhost-only and authenticated with a local client key. (#438)

## Documentation
- Updated the documentation set for the 2.0 runtime-rotation architecture, wrapper-first routing, app bind, storage paths, troubleshooting, and audit-archive guidance. (#438)
