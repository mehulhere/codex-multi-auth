# Desktop Capability Smoke Matrix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add and run a redacted, layered smoke suite that reports real evidence for every capability in the committed Codex Desktop feature profile.

**Architecture:** Define a typed capability manifest and dependency-injected probe runner in `lib/runtime/desktop-capability-smoke.ts`. Keep CLI parsing and human/JSON rendering in `scripts/test-desktop-capabilities.js`; deterministic tests use fake adapters, while `--live` invokes the installed Codex/router with disposable prompts and never prints credentials or generated bytes.

**Tech Stack:** TypeScript ESM, Node.js 18+, Fetch API, child-process spawning, Vitest, existing runtime/app-bind path helpers.

## Global Constraints

- Feature flags and installed files are discovery evidence, not live success.
- Never print access tokens, client secrets, account emails, prompts, images, or raw tool output.
- Live probes must be minimal and disposable.
- Computer Use clicks/typing and microphone speech remain explicit interactive boundaries.
- Unknown enabled Desktop features fail manifest completeness.
- Commit and push this harness before any WebSocket production code.
