# Audit Archive

Audit files in this directory are historical snapshots. They preserve the package version, commit, test output, and evidence captured at the time of the audit.

Do not treat old findings, test counts, dependency versions, or architecture diagrams in this directory as current source of truth. Use these current docs for the live architecture:

- [../architecture.md](../architecture.md)
- [../development/ARCHITECTURE.md](../development/ARCHITECTURE.md)
- [../configuration.md](../configuration.md)
- [../reference/commands.md](../reference/commands.md)

When a new audit is run, add a new dated artifact or update the audit heading with a clear snapshot note instead of rewriting captured evidence.
